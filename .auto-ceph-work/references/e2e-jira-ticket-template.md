# E2E Jira Ticket Template

`$auto-ceph-e2e`는 이 템플릿을 E2E 실행 기록용 Jira 티켓 description의 기준으로 사용한다.

```markdown
# [ACW E2E] <menu1> E2E 테스트

### E2E 테스트 정보
- 메뉴: <menu1>
- 타겟 케이스: <target-case-path>
- 실행 URL: <url>

### E2E 테스트 시나리오

#### 테스트 시나리오
1. `<url>`로 접속한다.
   - 기대결과: 로그인 화면이 표시된다.
2. `<id>`와 `<pw>`를 입력해 로그인한다.
   - 기대결과: 로그인에 성공하고 기본 화면으로 이동한다.
3. 선택 메뉴의 모든 기능 검증 절차를 작성한다.
   - 기대결과: 선택 메뉴의 기능 검증 기준을 만족한다.

#### 기대 결과
- 선택 메뉴의 주요 기능이 정상 동작한다.

#### 확인 범위
- 포함:
- 제외:

### E2E 테스트 결과
- 전체 결과: 시작 전
- 성공: 0/0
- 실패: 0/0
- 실패 케이스: 없음

### 작업 노트
- 티켓 시작 시간:
- 티켓 종료 시간:
```

규칙:

- 제목은 `[ACW E2E] <menu1> E2E 테스트` 형식이다.
- `### E2E 테스트 시나리오`는 `.auto-ceph-work/references/e2e-scenario-template.md` 구조를 따른다.
- `### E2E 테스트 시나리오`는 최초 실행 계획이며, E2E 실행 후에도 기존 numbered step 문장, 순서, 번호, 기능 설명은 수정하지 않는다.
- E2E 실행 후 parent skill은 각 numbered step 아래에 `기대결과`, `결과`, 실패 시 `실패 사유` annotation만 추가 또는 갱신한다.
- `### E2E 테스트 결과`는 통계 요약 전용 섹션이며, 전체 결과, 성공 수, 실패 수, 실패 케이스만 기록한다.
- `### 작업 노트`에는 `티켓 시작 시간`과 `티켓 종료 시간`을 기록한다.
